# Configuration — `.excalibur/`

Every Excalibur setting lives in plain files under `.excalibur/` at your repository root: human-readable, Git-versionable, optional. **Every command works without any configuration** — defaults apply, including the `standard-safe` safety preset.

## `config.yaml`

Generated by `excalibur init` with detected values only (commands are never invented):

```yaml
version: 1
project:
  name: quickcontract-api
  packageManager: pnpm
  languages: [typescript]
  frameworks: [nestjs]
commands:                 # top-level, canonical (project.commands is an accepted alias)
  test: pnpm test
  lint: pnpm lint
  typecheck: pnpm typecheck
  build: pnpm build
instructions:
  sources:                # detected instruction files (see ISD below)
    - path: ./CLAUDE.md
      format: claude_md
      scope: project
      enabled: true
safety:
  preset: standard-safe
workflowDefaults:         # command → workflow
  ask: ask-repo
  review: review-only
  patch: propose-patch
  run: standard-feature
  careful: structured-feature
  discovery: discovery
autonomyDefaults:         # command → autonomy level
  ask: 1
  review: 0
  patch: 2
  run: 3
  careful: 4
  discovery: 0
```

## Instruction & Skill Discovery (ISD)

Excalibur detects and uses the AI instruction files you already maintain — it never forces a rewrite into `.excalibur/`:

- **Project instructions** (trusted, used automatically): `CLAUDE.md`, `.claude/**`, `AGENTS.md`, `GEMINI.md`, `.cursor/rules`, `.github/copilot-instructions.md`, `.codex/**`, `.aider.conf.yml`, …
- **Project docs** (context): `README.md`, `CONTRIBUTING.md`, `docs/**`, ADRs.
- **Skills**: `**/SKILL.md` — detected but **never auto-enabled** when they need review.
- **User-global** (`~/.claude/**`): referenced locally only (`localOnly: true`), never copied into the repo without `--include-global`.

Manage them with:

```bash
excalibur instructions scan|list|inspect <id>|enable <id>|disable <id>|import <id>|doctor
excalibur skills list|inspect <id>|enable <id>|disable <id>
```

Enabling a `review_required` skill requires the explicit `--accept-risk` flag — `--yes` alone is never enough.

Before every model call, Excalibur builds the **effective instruction context**: enabled sources merged by precedence (enterprise policies → repo config → repo instructions → workflow instructions → user-global → docs → enabled skills), with per-source headers, secret redaction, conflict warnings and size caps. The exact context used is stored with every artifact as `effective-instructions.md`.

## Other files

| File / directory | Purpose |
|---|---|
| `extensions.yaml` | enable/disable extensions, register local ones |
| `models/providers.yaml` | model providers — env var **names** only, never keys |
| `models/routing.yaml` | model routing (role/path/workflow) |
| `instructions/*.md` | project instructions prepended to prompts |
| `workflows/`, `methodologies/`, `question-packs/`, `prompts/`, `artifacts/`, `policies/`, `reports/`, `roles/`, `command-mappings/` | declarative extension files that override/extend the built-ins |
| `runs/`, `patches/`, `interactions/`, `discovery/`, `reports/` | local artifacts |

## Init modes

- `excalibur init` — minimal (the files above, nothing else).
- `excalibur init --team` — adds `instructions/{architecture,testing,security}.md`, `policies/{standard-safe,sensitive-paths}.yaml`, `models/{providers,routing}.yaml` and asks whether to version `.excalibur/` in Git.
- `excalibur init --full` — exports all 14 workflows, 14 methodologies, question packs, prompts, artifact templates, policies, report templates, roles and command mappings for inspection and customization.

Validate any configuration with `excalibur doctor` and `excalibur extensions validate`.
